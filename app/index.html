Back to square one

I find myself refering back to my 2022 version of shortstack OpenProps to retred previously burn ground.

Instantly I have to fight Adam Argyle’s too smart by half layout. His layout is a grid layout in columns – with the header on the left, next body, next footer.

It is within an @media query landscape declatation. Why anyone would want to put that type of layout – or ANY type of layout in a landscape media query is beyond me.

Adam Argyle releases OpenProps as a minified, unreadable npm module. In order to get unminified CSS it is necesary to go to the OpenProps Github repo, download it and extract the source files.

I’ve added my own CSS file at the bottom of the index.css file as a new pair of glasses. I will need to start by undoing, correcting, eleminating his opinionated property declarations.

Now that I have that installed I don’t need the NPM module but I will have to look and occaisionaly update it. Or not.
